Documentation

The database have 6 properties
1. name
2. price
3. featured
4. rated
5. createdAt
6. company

Filters

After the root url you can use a filter by supplying the ? and a key value pair.

1.https://storeapi-0t62.onrender.com/api/v1/products?name=sofa will return all the products with name sofa in it.
2. https://storeapi-0t62.onrender.com/api/v1/products?featured=true will return all the feature products
3. https://storeapi-0t62.onrender.com/api/v1/products?name=sofa&company=ikea will return the products which have sofa in name and ikea is their company
4. https://storeapi-0t62.onrender.com/api/v1/products?name=sofa&fields=companywill return only the company values not other details.
5. Add as many filters like that with & to append them.



Sorting and pagination

1. https://storeapi-0t62.onrender.com/api/v1/products?sort=price return the sorted products. Use -price to get descending order results.

2. https://storeapi-0t62.onrender.com/api/v1/products?page=1&limit=3 return the divide the products in pages where each page will have 3 products and it will return the 1st page. by Default the limit is 10 and page is 1.



Numeric Filters

1. https://storeapi-0t62.onrender.com/api/v1/products?price>200&sort=-price will filter the data with all product of price greater than 200 and then it will sort the results in desending order.
